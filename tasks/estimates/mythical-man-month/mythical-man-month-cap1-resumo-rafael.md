##Parte 1

	O desenvolvimento de grandes sistemas de software é tão difícil como foi
 para os grandes e pesados animais da pré-história libertarem-se de um poço
 de piche, muitos sucumbiram e foram tragados.

“É necessário 3 vezes mais esforço para desenvolver um componente de software
 em relação a um software para uso privado, assim como 3 vezes mais esforço 
para transformá-lo em um produto coerente e testado.”

	Isso quer dizer que gerar um script para uso pessoal é mais rápido e mais fácil
 do que gerar para uso público ou comercial como um produto, pois vai precisar
 modifica-lo para atender casos mais genéricos e parametrizar as informações 
que era fixas em um ambiente. Transformar isso em uso em qualquer interessado
 em outras empresas como produto vai demandar muito mais esforço.

	Movimento como agile software crafsmanship reforçam a ideia de que a profissão
de programar envolve gratificação pessoal: a alegria de construir as coisas

	Brooks afirma que programadores precisam se adaptar a perfeição exigido. Programadores
dependem de outros e as vezes não tem autoridade necessária para fazer as coisas acontecerem.

	Quando temos várias ideias, mas o trabalho para colocar tudo em prática nos desestimula, pois
para cada momento para projetar algo é acompanhado por muitas horas de trabalho desgastante para atingir os objetivos.

	Na medida que chega ao fim o projeto de software converge mais devagar, pois muitas atividades ocultas são descobertas,
bugs se acumulam e a base de código e a complexidade crescem e a experiência não ajudar nesse problema.
